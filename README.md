# ISL Goa Sign Language Recognition

A PyTorch-based sign language recognition system for the ISL (Indian Sign Language) Goa dataset. This project uses MediaPipe landmarks and a Transformer-based classifier to recognize sign language gestures from video data.

## Overview

This project provides a complete pipeline for:
- Extracting one-hand keypoints from sign language videos using MediaPipe Holistic and flipping to right-handedness
- Training a Transformer / LSTM classifier on landmark sequences
- Evaluating model performance on test data

The system focuses on right-hand landmarks (or mirrored left-hand when right is unavailable).

## Data Format

### Input Videos
- MP4 format video files
- Organized in train/val/test splits -> each split contains User_00X/LABEL__...mp4 format paths
- Filename format: `LABEL__*.mp4` or `LABEL_*.mp4` (label extracted from prefix)

### Landmarks (H5 Files)
- Generated by `generate_landmarks.py`
- Dataset key: `'intermediate'`
- Shape: `(T, 1, 21, 3)` where:
  - `T` = number of frames
  - `21` = MediaPipe hand landmarks
  - `3` = x, y, z coordinates

## Usage

### 1. Extract Landmarks from Videos

```bash
python generate_landmarks.py \
    --input-dir /path/to/video/split \
    --output-dir /path/to/h5/split \
    --input-features 21
```

This will process all MP4 files in the input directory and create corresponding H5 files with MediaPipe hand landmarks.

### 2. Train the Model

Update the paths in `runner.py`:

```python
DATA_ROOT = "/data/ISL_GOA_SAMPLE_h5"
TRAIN_DIR = os.path.join(DATA_ROOT, "train_split")
VAL_DIR   = os.path.join(DATA_ROOT, "val_split")
TEST_DIR  = os.path.join(DATA_ROOT, "test_split")
```

Then run:

```bash
python runner.py
```

### Configuration

Key hyperparameters in `runner.py`:

```python
seed = 43               # Random seed
max_frames = 200        # Maximum sequence length
batch_size = 64         # Training batch size
epochs = 100            # Maximum training epochs
patience = 6            # Early stopping patience
lr = 5e-4              # Learning rate
weight_decay = 1e-3    # L2 regularization
```

## Training Features

- **Data Augmentation**: Random temporal cropping and frame subsampling
- **Normalization**: Per-landmark z-score normalization centered on wrist
- **Optimization**: AdamW with learning rate scheduling (ReduceLROnPlateau)
- **Mixed Precision**: Automatic mixed precision for faster training
- **Early Stopping**: Monitors validation accuracy with patience
- **Logging**: Detailed training logs and JSON summaries

## Output

Training produces:
- `logs/{run_name}/`: Directory with all outputs
- `{run_name}.log`: Detailed training log
- `summary.json`: Final results and configuration

## Metrics

- Training/validation accuracy and loss per epoch
- Per-class precision, recall, F1-score
- Confusion matrix
